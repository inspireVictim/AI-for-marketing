## Telegram Lead Qualification Bot (MLM, RAG + LangChain)

Этот проект — Telegram-бот для квалификации лидов в сетевом маркетинге (MLM).  
Он использует **LLM + RAG по маркетинговым книгам** для того, чтобы **отсеивать нерентабельных клиентов** и передавать только адекватных и перспективных лидов администратору/основателю.

Архитектура построена так, чтобы **логика ИИ была полностью отделена от Telegram-части**.

---

### Стек

- **Python** 3.11+
- **aiogram** 3.x — Telegram-бот (асинхронный стек)
- **LangChain** — оркестрация LLM, RAG-пайплайн
- **ChromaDB** — векторная база для поиска по PDF-книгам
- **OpenAI / OpenRouter совместимый API** — языковая модель и эмбеддинги
- **Docker** — упаковка и деплой

---

### Логика воронки и перевёрнутая воронка

1. **Широкий вход (Lead Magnet)**  
   Пользователь получает **бесплатную консультацию от ИИ-эксперта по маркетингу**: бот даёт пользу и советы, опираясь на книги из базы знаний (RAG по PDF).

2. **Профилирование (Profiling)**  
   Бот задаёт **3–4 ключевых вопроса**:
   - текущий опыт в MLM / маркетинге / бизнесе;
   - наличие и порядок бюджета на трафик;
   - сколько времени человек готов уделять внедрению;
   - цели и ожидания.

3. **Скоринг (Scoring)**  
   ИИ анализирует ответы и данные профиля:
   - если по ходу диалога выясняется, что **бюджет < $100 и нет опыта**, бот:
     - вежливо завершает диалог,
     - даёт ссылку на обучающий канал и/или базовый PDF-гайд для прогрева;
   - если клиент выглядит **рентабелен и осознан**, бот:
     - продолжает диалог как опытный маркетолог,
     - запрашивает контакт (телефон / Telegram / удобный способ связи),
     - передаёт данные админу.

4. **Результат**  
   - **Нерентабельный**: получает автоматический совет, возможно PDF-гайд и ссылку на обучающий канал; уходит в «прогрев».  
   - **Рентабельный**: получает приглашение на созвон (дальше можно интегрировать Google Calendar); его данные уходят админу.

Вся эта бизнес-логика **зашита в системный промпт** в `agent.py` и может быть дополнительно уточнена.

### Пошаговый сценарий внедрения

#### Шаг 1: Загрузка базы знаний (индексация PDF в ChromaDB)

**Задача:**  
«Напиши скрипт на Python с использованием LangChain и ChromaDB, который загружает все PDF-файлы из папки `/books`, разбивает их на чанки по 1000 символов и сохраняет в векторную базу данных».

В проекте эта логика реализована в `database.py`:

- загрузка всех PDF из директории `KNOWLEDGE_DIR` (по умолчанию `./books`);
- нарезка на чанки по ~1000 символов с overlap;
- сохранение в ChromaDB.

Запуск индексации:

```bash
python database.py
# или с принудительной пересборкой
FORCE_REBUILD=true python database.py
```

#### Шаг 2: «Агент-Квалификатор» (асинхронный Telegram-бот + RAG)

**Задача:**  
«Создай асинхронного Telegram-бота на aiogram. Интегрируй его с моей векторной базой данных. Бот должен вести диалог как опытный маркетолог. Если в ходе диалога выясняется, что у клиента бюджет < $100 и нет опыта, бот должен вежливо дать ссылку на обучающий канал. Если клиент рентабелен — бот должен запросить его контакт и отправить уведомление админу».

В текущем проекте:

- `main.py` — поднимает асинхронный aiogram-бот, подключает роутеры и запускает long polling;
- `handlers.py`:
  - обрабатывает `/start` и текстовые сообщения;
  - передаёт текст пользователя в `LeadQualificationAgent`;
  - отправляет обратно сгенерированный ответ;
  - если лид признан рентабельным — шлёт уведомление админу (`ADMIN_CHAT_ID`);
- `agent.py`:
  - асинхронный метод `process_message(user_id, user_text)`:
    - подтягивает контекст из Chroma (через retriever);
    - использует системный промпт с бизнес-логикой переводёрнутой воронки и стилем общения «опытного маркетолога»;
    - анализирует опыт, бюджет, мотивацию и цели;
    - возвращает строгий JSON (`qualified`, `reason`, `reply_to_user`, `name`, `experience`, `budget`, `goals`);
  - жёстко зашито правило: при бюджете < $100 и отсутствии опыта + отсутствующей готовности учиться — лид считается нерентабельным, а в ответе формируется мягкий отказ + рекомендации по обучающему каналу/гайду.

#### Шаг 3: Интеграция с вашей MLM-воронкой

На этом шаге бот начинает «знать» особенности именно вашей модели и монетизации.

- **Prompt Engineering (MLM-специфика)**:  
  в системный промпт агента (`agent.py`) вшито:

  > «Твоя цель — не продать в лоб, а выявить амбициозных лидеров. Используй методики из книги \[Название\] для проверки мотивации.»

  Это фиксирует для модели:

  - что задача — не «дожимать» слабых, а **находить сильных**;
  - что при оценке нужно опираться на конкретные методики мотивационной диагностики (из вашей книги).

- **Модель монетизации и фильтрации**:  
  - **Нерентабельный**:
    - получает **краткий план действий** и/или **PDF-гайд**;
    - получает **ссылку на обучающий канал** (`TRAINING_CHANNEL_URL`);
    - уходит в «прогрев», без вовлечения вашего времени.
  - **Рентабельный**:
    - получает приглашение на **созвон**;
    - его данные отправляются админу (`ADMIN_CHAT_ID`);
    - на следующем шаге можно интегрировать Google Calendar.

### Установка и запуск (без Docker)

1. **Перейдите в директорию проекта**

```bash
cd AIMarketing
```

2. **Создайте виртуальное окружение и активируйте его**

```bash
python3.11 -m venv .venv
source .venv/bin/activate
```

3. **Установите зависимости**

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

4. **Создайте файл `.env` на основе `.env.example` (см. в README или скопируйте вручную)**  
Пример содержимого:

```env
BOT_TOKEN=your_telegram_bot_token_here

OPENAI_API_KEY=your_openai_or_openrouter_key_here
OPENAI_BASE_URL=

OPENAI_CHAT_MODEL=gpt-4.1-mini
OPENAI_EMBEDDINGS_MODEL=text-embedding-3-large

ADMIN_CHAT_ID=123456789

KNOWLEDGE_DIR=./books
CHROMA_PERSIST_DIR=./chroma_db

TRAINING_CHANNEL_URL=https://t.me/your_training_channel
TRAINING_GUIDE_URL=https://example.com/your_guide.pdf
```

5. **Подготовьте папку с книгами**

```bash
mkdir -p books
cp /path/to/your/books/*.pdf books/
```

6. **Проиндексируйте знания в ChromaDB (Шаг 1)**

```bash
python database.py
```

7. **Запустите Telegram-бота (Шаг 2)**

```bash
python main.py
```

### Запуск через Docker

1. **Соберите образ**

```bash
docker build -t aimarketing-bot .
```

2. **Создайте `.env` рядом с Dockerfile** (по примеру выше).

3. **Первый запуск — индексация базы знаний**

```bash
docker run --rm \
  --env-file .env \
  -v $(pwd)/books:/app/books \
  -v $(pwd)/chroma_db:/app/chroma_db \
  aimarketing-bot \
  python database.py
```

4. **Запуск бота**

```bash
docker run --rm \
  --env-file .env \
  -v $(pwd)/books:/app/books \
  -v $(pwd)/chroma_db:/app/chroma_db \
  aimarketing-bot
```

### Как работает фильтр (технически)

- `agent.py`:
  - создаёт `ChatOpenAI` с моделью из `OPENAI_CHAT_MODEL`;
  - подключает Chroma через `database.get_vectorstore()` и строит retriever;
  - хранит краткую историю диалога по каждому `user_id` в памяти;
  - собирает системный промпт с бизнес-логикой и MLM-спецификой (перевёрнутая воронка, бюджет, опыт);
  - при наличии добавляет RAG-контекст (фрагменты книг);
  - ожидает **строго JSON**-ответ и парсит его в `LeadData`.

- `handlers.py`:
  - обрабатывает `/start` и текстовые сообщения;
  - на любое текстовое сообщение:
    - вызывает `agent.process_message(user_id, text)`;
    - отправляет пользователю `lead.reply_to_user`;
    - если `lead.qualified == True` — формирует компактное сообщение и отправляет его в `ADMIN_CHAT_ID`.

- `database.py`:
  - загружает все PDF из `KNOWLEDGE_DIR` (`./books` по умолчанию);
  - режет их на чанки по 1000 символов;
  - строит или подключает ChromaDB (`CHROMA_PERSIST_DIR`).

---

### Кастомизация

- Жёсткость фильтра (минимальный бюджет, требования к опыту) задаётся в системном промпте `agent.py`.
- Ссылки на обучающий канал и гайд управляются через ENV (`TRAINING_CHANNEL_URL`, `TRAINING_GUIDE_URL`).
- Расположение книг и базы знаний регулируется переменными `KNOWLEDGE_DIR` и `CHROMA_PERSIST_DIR`.
